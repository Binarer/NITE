# Техническое задание (ТЗ) на разработку мобильного приложения "NiTe"

## 1. Общая информация

### 1.1 Название проекта
**NiTe** — мобильное приложение для управления задачами (таск-трекер) с акцентом на еженедельное планирование и продвинутую категоризацию.

### 1.2 Цель создания
Разработать современное, стильное и функциональное приложение для планирования недельных задач с возможностью детальной настройки, drag-and-drop интерфейсом, интеграцией с AI для оценки приоритетов и ведением базы продуктов (для категории "Еда").

### 1.3 Платформа
- Мобильное приложение: **Android / iOS**
- Фреймворк: **Flutter (Dart)**
- UI библиотека: **GetWidget** (для ускорения разработки UI с готовыми компонентами)

### 1.4 Целевая аудитория
Пользователи, ведущие активный образ жизни, желающие структурировать свои недельные задачи: тренировки, питание, работа, учеба и личные дела.

## 2. Функциональные требования

### 2.1 Дизайн и стилистика (UI/UX)
- **Цветовая гамма:** Черный фон (`#000000` или близкий к черному), серые акценты (для карточек, текста, границ). Белый текст для контраста.
- **Стиль:** Современный (Neomorphism / Glassmorphism с серыми оттенками), минималистичный, акцент на контенте.
- **Анимация:** Плавные переходы между экранами, анимация drag-and-drop, появление модальных окон.

### 2.2 Основные сущности

#### 2.2.1 Теги (Категории задач)
- Каждая задача может иметь один или несколько тегов.
- Системные теги по умолчанию: `Тренировка`, `Еда`, `Работа`, `Задача`, `Учеба`.
- **Атрибуты тега:**
    - Название (string)
    - Смайлик/Эмодзи (string, выбор из стандартной библиотеки или кастомный)
    - Цвет (Color, для подсветки задач в списке)

#### 2.2.2 Задача (Task)
- **Название** (string, обязательное)
- **Описание** (string, опционально)
- **Теги** (List<Tag>, множество)
- **Приоритет** (int, от 0 до 5). Где 0 - нет приоритета, 5 - максимальный.
- **Флаг "Оценка от AI"** (bool). Если `true`, приоритет рассчитывается/корректируется через Mistral AI.
- **Время выполнения** (опционально):
    - Время начала (часы:минуты)
    - Время конца (часы:минуты)
- **Дата выполнения** (Date). Задача привязана к конкретному дню недели.
- **Сценарий** (foreign key to Scenario). Задача может быть частью сценария.

#### 2.2.3 Категория "Еда" (Food Item)
Специализированная сущность, привязываемая к задачам с тегом "Еда".
- **Название блюда/продукта** (string)
- **Фотография** (путь к файлу/URL)
- **Описание** (string)
- **Калорийность (ккал)** (int/float)
- **БЖУ** (struct):
    - Белки (граммы)
    - Жиры (граммы)
    - Углеводы (граммы)
- **Связь:** При создании задачи с тегом "Еда" пользователь может выбрать созданную карточку еды из библиотеки, либо создать новую "быструю" задачу без привязки к карточке.

#### 2.2.4 Сценарии на неделю (Scenario)
Возможность создавать шаблоны задач на неделю для быстрого заполнения.
- **Название сценария** (string), например "Спортивная неделя", "Рабочая неделя".
- **Список задач-шаблонов**, где у задач указаны:
    - Название, описание, теги, время (часы:минуты), приоритет (по умолчанию), флаг "Оценка от AI".
    - *Важно:* Задачи в сценарии не привязаны к конкретной дате, а привязаны к дню недели (ПН, ВТ, ...).
- **Применение сценария:** При выборе сценария система создает копии задач-шаблонов на соответствующие дни текущей или следующей недели.

### 2.3 Основные экраны и логика

#### 2.3.1 Главный экран (Еженедельник)
- Отображение текущей недели (ПН-ВС).
- Сетка: Дни недели (колонки) / Временные слоты (строки, если у задачи есть время).
- **Drag and Drop:**
    - Возможность перемещать задачи между днями недели.
    - Возможность менять время задачи (перетаскивание внутри временной шкалы).
    - Возможность менять порядок задач в пределах одного дня.
- **Отображение задач:**
    - Карточка задачи отображает: название, иконки тегов (смайлики), цветной индикатор приоритета.
    - Приоритет визуализируется (например, цветом левой границы карточки или уровнем прозрачности/яркости).
- **Кнопка "Добавить задачу"** (ведет на экран создания/редактирования).

#### 2.3.2 Экран создания/редактирования задачи
- Поля: Название, Описание.
- Выбор тегов (из существующих, с возможностью перехода к созданию нового тега).
- Выбор даты и времени (Start / End time).
- Ползунок (Slider) или выбор цифрами приоритета от 0 до 5.
- **Чекбокс "Оценить приоритет через AI"** (Mistral API).
    - Если чекбокс включен, после нажатия "Сохранить", происходит вызов API, полученный приоритет (0-5) проставляется в поле приоритета, но пользователь может его изменить вручную (переопределить).
- Кнопка "Привязать карточку еды" (активна только если выбран тег "Еда"). Открывает список существующих Food Item для выбора или кнопку создания нового.

#### 2.3.3 Экран "Библиотека продуктов (Еда)"
- Список всех созданных карточек еды (фото + название + калории).
- Поиск по названию.
- Кнопка "Добавить продукт" (ведет на экран создания карточки еды).
- При нажатии на карточку — детальный просмотр (Фото, Название, Описание, БЖУ, Калории), возможность редактирования.

#### 2.3.4 Экран "Сценарии"
- Список сохраненных сценариев.
- Кнопка "Создать сценарий" (ведет в конструктор сценария).
- **Конструктор сценария:**
    - Ввод названия сценария.
    - Интерфейс планировщика на неделю (ПН-ВС), где можно добавлять задачи-шаблоны (без конкретной даты, но с привязкой ко дню недели и времени).
- Применение сценария (кнопка "Применить на эту неделю" / "Применить на следующую неделю").

#### 2.3.5 Экран настроек (Settings)
- Настройка часового пояса (по умолчанию Екатеринбург, но лучше определять системный).
- Управление тегами (добавление, удаление, изменение смайлика/цвета).
- Настройки уведомлений (оценка за неделю, напоминания о задачах).
- API ключ для Mistral (или использование встроенного ключа приложения).

### 2.4 Интеллектуальные функции (Mistral AI)

#### 2.4.1 Оценка приоритета задачи
- **Триггер:** Пользователь создает/редактирует задачу и включает чекбокс "Оценить приоритет через AI".
- **Механика:** При сохранении задачи формируется промпт для Mistral AI, содержащий: название задачи, описание, выбранные теги, время (если есть), возможно, контекст других задач на этот день.
- **Ответ AI:** Число от 0 до 5.
- **Действие:** Полученное число подставляется в поле "Приоритет". Пользователь видит это и может изменить (т.е. AI дает рекомендацию).

#### 2.4.2 Еженедельная оценка (Ретроспектива)
- **Триггер:** Каждый понедельник в 12:00 по времени Екатеринбурга (GMT+5).
- **Механика:**
    1.  Система собирает данные по выполненным задачам за прошлую неделю (те задачи, которые пользователь отметил как выполненные, либо просто завершившиеся по времени).
    2.  Формируется промпт для Mistral AI на основе: списка выполненных задач, их приоритетов, тегов (например, "На прошлой неделе выполнено: 3 задачи 'Работа' с приоритетом 5, 2 задачи 'Тренировка' с приоритетом 4... Дай краткую оценку продуктивности за неделю и совет на следующую").
    3.  **Вывод:** Приходит push-уведомление с текстом "Ваша недельная статистика: [краткая оценка от AI]". В приложении появляется pop-up или специальный экран с развернутым отчетом.

### 2.5 Локальное хранение данных
- Использовать **Hive** или **SQFlite** для локального хранения всех данных (задачи, теги, сценарии, карточки еды), так как приложение не требует серверной части.
- Для хранения изображений (фото еды) использовать локальное хранилище устройства.

## 3. Нефункциональные требования

### 3.1 Производительность
- Плавный скролл списков (виртуализация).
- Оптимизация работы с изображениями (кэширование, сжатие).
- Drag-and-drop должен работать без задержек.

### 3.2 Платформенные особенности
- Корректное отображение на различных размерах экранов (адаптивный дизайн).
- Поддержка темной темы (у нас и так черный фон, так что ок).
- Поддержка локальных уведомлений (flutter_local_notifications).

### 3.3 Интеграции
- **Mistral AI API:** HTTP клиент (Dio или http) для отправки запросов и получения ответов. Необходимо обрабатывать ошибки сети и таймауты.
- Работа с камерой/галереей (image_picker) для загрузки фото еды.

## 4. Стек технологий

- **Frontend:** Flutter (SDK >= 3.0)
- **UI Kit:** GetWidget (использовать готовые компоненты: `GetButton`, `GetCard`, `GetListTile`, `GetTextField` и т.д., кастомизируя их под черно-серую гамму).
- **State Management:** GetX (так как GetWidget с ним хорошо интегрируется) или Riverpod / BLoC (на усмотрение разработчика, но GetX проще).
- **Local DB:** Hive (простота, скорость) или SQFlite.
- **Local Notifications:** `flutter_local_notifications`.
- **Image Picker:** `image_picker`.
- **Drag and Drop:** Встроенные виджеты Flutter (`LongPressDraggable`, `DragTarget`) или пакеты типа `reorderables`.
- **Date/Time Handling:** `intl` для форматирования, работа с часовыми поясами.


### Примерное описание главного экрана:
- Сверху: Название "NiTe", кнопка "Сценарии", кнопка "Настройки".
- Ниже: Полоса с днями недели (ПН, ВТ, ... ВС) с возможностью свайпа по горизонтали для переключения недель.
- Основная область: Двумерная прокрутка (горизонтальная по дням, вертикальная по времени).
- Карточка задачи: Черный фон, серая обводка, слева цветная полоска (приоритет), текст белый, справа ряд эмодзи (теги).
- Снизу: Кнопка "+" (добавить задачу).

## 6. Этапы разработки (Roadmap)

1.  **Этап 1:** Настройка проекта, подключение зависимостей, создание моделей данных (Task, Tag, FoodItem, Scenario).
2.  **Этап 2:** Разработка главного экрана (еженедельник) и логики drag-and-drop. Работа с локальной БД (сохранение/загрузка задач).
3.  **Этап 3:** Реализация CRUD для задач и тегов. Экран создания задачи.
4.  **Этап 4:** Разработка функционала "Еда" (карточки продуктов, привязка к задачам).
5.  **Этап 5:** Разработка функционала "Сценарии".
6.  **Этап 6:** Интеграция с Mistral AI (оценка приоритета, еженедельная ретроспектива + push-уведомления).
7.  **Этап 7:** Настройки, финальная кастомизация UI под черно-серый стиль, тестирование, исправление багов.

## 7. Примечания
- Время в приложении по умолчанию — Екатеринбург (GMT+5). Желательно предусмотреть автоопределение, но с возможностью ручной настройки.
- Все тексты в интерфейсе должны быть на русском языке (локализация на английский не требуется на старте).
- Приложение должно работать полностью офлайн. AI-функции требуют интернета, при его отсутствии чекбокс "Оценка от AI" должен быть неактивен (disabled) с подсказкой.
